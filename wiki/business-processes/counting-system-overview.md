# Counting System Architecture

Overview of the automated, five-phase counting process that maintains inventory accuracy.

## System Overview

The Clesen Horticulture counting system is a fully automated, five-phase process that uses intelligent, business-driven cycle counting to maintain accurate inventory without requiring full physical audits.

---

## Five-Phase Process

### Phase 1: Preparation (Automated)

**Codeunit:** 50108 CLE Prepare Auto Item Count  
**Schedule:** Every 2 hours  
**What Happens:**
- Evaluates 9 count type rules
- Creates count lines based on business logic
- Sets status to "Prepared"
- Assigns to appropriate locations/users
- Calculates priority scores

**Purpose:** Automatically identify which items need counting based on business rules (shipments, production, negative availability, etc.)

### Phase 2: Release (Automated)

**Codeunit:** 50115 CLE Release Count  
**Schedule:** Every 1 minute  
**What Happens:**
- Validates count lines can be released
- Checks bin existence
- Prevents duplicate counts
- Delays release if zone is being picked
- Delays release if purchase receipts expected
- Changes status from "Prepared" to "Released"
- Makes visible to warehouse staff

**Purpose:** Make counts visible to warehouse staff at the optimal time (when picking isn't active, when receipts won't affect accuracy)

### Phase 3: Counting (Manual)

**Page:** 50125 Inventory Count Lines  
**Performed by:** Warehouse Staff  
**What Happens:**
- Staff see released lines on count sheet
- Physical counting performed in the warehouse
- Quantity entered in system
- "Finish & Submit" changes status to "Counted"
- Deviation warnings trigger if count far from expected

**Purpose:** Capture actual physical inventory quantities

### Phase 4: Processing (Automated)

**Codeunit:** 50111 CLE Process Incoming Counts  
**Schedule:** Every 1 minute  
**What Happens:**
- Calculates quantity adjustment needed
- Evaluates deviation from expected quantity
- **Small deviation (acceptable):** Posts adjustment automatically, archives line
- **Medium deviation:** Requests recount (status back to "Prepared")
- **Large deviation:** Requests supervisor assistance

**Purpose:** Automatically resolve minor discrepancies and flag significant issues for manual review

### Phase 5: Exception Handling (Manual)

**Page:** 50127 Inventory Count Posting  
**Performed by:** Supervisors/Managers  
**What Happens:**
- Review high-deviation counts
- Approve adjustment posting
- Request additional recount
- Adjust acceptable limits
- Resolve assistance requests

**Purpose:** Human review and approval of significant inventory adjustments

---

## Status Lifecycle

```
Prepared → Released → Counted → Archived
    ↑                   ↓
    └─── Recount Request (medium deviation)
         ↓
         Assistance Required (large deviation)
```

**Status Definitions:**

| Status | Meaning | Phase |
|--------|---------|-------|
| Prepared | Count line created, waiting for release validation | 1 |
| Released | Available to warehouse staff for counting | 2 |
| Counted | First count submitted, awaiting processing | 3 |
| Recount Request | Medium deviation, needs second count | 4 |
| Recounted | Second count submitted | 4 |
| Assistance Required | Large deviation, needs supervisor approval | 4 |
| Archived | Complete, adjustment posted | 5 |

## Count Type Matrix

The system automatically creates counts based on 9 business rules:

| Count Type | Trigger | Frequency | Business Purpose |
|------------|---------|-----------|------------------|
| 2 Weeks Ship | Sales orders due in 14 days | Every 2 hours | Prevent shipment surprises |
| Ready Count | Production orders 50%+ complete | Every 2 hours | Catch overproduction/scrap |
| Sowing Check | 3 days after sowing | One-time per order | Verify germination rates |
| Transplant Check | 7 days after transplant | One-time per order | Verify transplant survival |
| Negative Availability | Negative inventory/available | Every 2 hours | Fix data integrity issues |
| High Demand | High sales volume | Every 2 hours (optional) | Prevent popular item stockouts |
| Physical Inventory | Manual trigger | On-demand | Full audit, financial compliance |
| Location Count | Manual trigger | On-demand | Zone audit, reorganization |
| Single Item | Manual trigger | On-demand | Investigate specific discrepancies |

## Line Source Types

Count lines originate from three sources:

1. **Automatic Count Rules:** Lines generated by the 9 business rules (99% of counts)
2. **Manual Physical Inventory:** Staff-initiated full facility count
3. **Manual Location Count:** Staff-initiated count of specific location or zone

---

## Deviation Thresholds

The system evaluates count deviations and determines action based on configurable thresholds:

**Small Deviation (Acceptable):**
- Quantity within acceptable tolerance
- Adjustment posted automatically
- No recount requested
- Line archived

**Medium Deviation (Recount):**
- Quantity exceeds small deviation but below large deviation threshold
- Recount requested (status back to "Prepared")
- Staff performs second count
- If second count matches first, archives
- If second count differs significantly, escalates to assistance

**Large Deviation (Assistance):**
- Quantity significantly differs from expected
- Status set to "Assistance Required"
- Supervisor must review
- Supervisor approves adjustment or requests investigation
- Line archived after approval

## Key Benefits

✅ **Continuous Accuracy:** Counts run automatically throughout the day
✅ **Risk-Based Prioritization:** Items needing count most urgently are prioritized first
✅ **Automated Decision-Making:** Small discrepancies resolved without human intervention
✅ **Compliance Ready:** Full audit trail and adjustment history maintained
✅ **Flexible Thresholds:** Deviations adjusted per item type or location
✅ **No Full Shutdowns:** Inventory accuracy maintained without facility shutdowns

---

## Integration Points

The counting system integrates with:

- **Availability Management:** Counts trigger based on availability thresholds
- **Production Orders:** "Ready Count" checks production progress
- **Sales Orders:** "2 Weeks Ship" ensures shipment stock
- **Purchase Receipts:** Release logic delays counts during receipts
- **Picking Operations:** Release logic waits for picking zones to clear
