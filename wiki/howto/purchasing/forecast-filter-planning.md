# Forecast Filter — Planning Worksheet

Streamlined approach to removing off-season planning lines from the Planning Worksheet after plan calculation.

## What This Is

When you run **Calculate Regenerative Plan** on the Planning Worksheet with a Demand Forecast selected, Business Central recalculates supply for every item in the system—not just items in your forecast. This populates the worksheet with planning lines for Spring and Summer items even when planning for Fall, making it difficult to focus on relevant items.

The **Forecast Filter** solves this by providing an **Apply Forecast Filter** button. After the plan calculation finishes, you click the button and confirm—all planning lines for items with no quantity in the configured forecast are removed, leaving only lines relevant to your current planning season.

### Key Concepts

| Concept | Definition |
|---------|-----------|
| **Demand Forecast** | Named forecast record holding projected demand quantities by item and date |
| **Calculate Regenerative Plan** | Planning Worksheet action that recalculates all supply requirements from scratch |
| **Planning Line** | Single row on the Planning Worksheet representing a suggested purchase or production order for one item |
| **Apply Forecast Filter** | Button in the Process menu that removes all MPS lines for items with no quantity in the configured forecast |
| **Production Forecast (batch field)** | Field on planning worksheet batch (in Req. Wksh. Names) that tells the filter which forecast to check against |

## When to Use It

Use the Forecast Filter every time you run Calculate Regenerative Plan with a seasonal forecast:

- **Switching seasons** — When transitioning from Summer to Fall, Calculate Plan populates the worksheet with all items. Click **Apply Forecast Filter** to reduce to Fall items only.
- **Daily planning reviews** — After any regenerative calculation where you want to work only within the active forecast scope.
- **Before converting lines to orders** — Filter down to relevant lines first to avoid accidentally creating purchase or production orders for off-season items.

## Setup

Configure which forecast your planning batch should check against. This is a one-time setup per batch—only update when switching seasonal forecasts.

1. Search for **Req. Wksh. Names** in Business Central
2. Find the batch you use (for example, **DEFAULT**)
3. Set the **Production Forecast** field to the forecast you plan against (for example, **FALL-2026**)
4. Close the page (saved automatically)

**Note:** For day-to-day use within the same season, the setting persists—no need to update it unless changing to a different forecast.

## Before You Start

You need:
- Access to **Planning Worksheet** in Business Central
- **Production Forecast** field set on your batch in **Req. Wksh. Names** (if blank, Apply Forecast Filter will error)
- A Demand Forecast with quantities entered for current season items (at least one item with quantity > 0)
- Standard planning worksheet permissions

**Note:** Contact your system administrator if Req. Wksh. Names is not accessible or you lack permission to run Calculate Regenerative Plan.

## How to Use the Forecast Filter

### Step 1: Confirm Your Batch Has the Forecast Configured

Before running the calculation, verify the **Production Forecast** field is set in **Req. Wksh. Names** (see Setup above). If already done for the current season, no action needed—the setting persists.

### Step 2: Run Calculate Regenerative Plan

1. Open the **Planning Worksheet** in Business Central
2. Click **Actions** in the top menu
3. Select **Functions** → **Calculate Regenerative Plan**
4. Set planning dates and other parameters as usual
5. Click **OK**

Wait for the calculation to finish (time varies by item count).

### Step 3: Click Apply Forecast Filter

After Calculate Regenerative Plan finishes, click **Apply Forecast Filter** in the Process menu.

A confirmation dialog appears:

> "Forecast 'FALL-2026' is set on this batch. Do you want to remove planning lines for items that have no quantity in this forecast?"

Choose:

| Choice | What Happens |
|--------|-------------|
| **Yes** | All MPS planning lines for items with no forecast quantity are deleted. A summary confirms how many lines removed and remain. |
| **No** | No changes. All planning lines generated by the calculation remain. |

### Step 4: Review the Summary (Yes Path)

After clicking **Yes**, a confirmation message appears, for example:

> "14 line(s) removed. 38 line(s) remain."

The worksheet now shows only items with positive quantity in the selected forecast. Review, adjust, and convert lines to orders as normal.

**Example — Fall Planning Scenario:**

| Field | Value | Notes |
|-------|-------|-------|
| Batch Production Forecast | FALL-2026 | Set in Req. Wksh. Names |
| Lines before filter | 52 | All items across all seasons |
| Lines removed | 14 | Spring/Summer items with no Fall qty |
| Lines remaining | 38 | Fall items ready for review |

## What Gets Removed and What Stays

### Lines That Are Removed (When Yes Is Clicked)

A planning line is removed when the item has **no forecast entry with quantity > 0** in the configured forecast:

- Item has no entries in the selected forecast, **OR**
- All forecast entries for the item have quantity ≤ 0

Only **MPS lines** (production order suggestions for finished goods) are evaluated. MRP lines (component purchase order suggestions) are never removed.

### Lines That Stay

A planning line stays when the item has **at least one forecast entry with quantity > 0** in the selected forecast, regardless of date or location.

### Lines Never Affected

- The filter only removes lines after you click **Yes** in confirmation dialog. Clicking **No** leaves worksheet entirely unchanged.
- **MRP lines** (component purchase order suggestions) are always kept regardless of forecast.
- Lines added manually to the worksheet are evaluated the same as calculated lines (by item number vs. forecast entries).

## Troubleshooting

### Issue 1: "No forecast is set on this worksheet batch"

**Cause:** The **Production Forecast** field on your planning batch (in Req. Wksh. Names) is blank.

**Solution:**
1. Search for **Req. Wksh. Names** and open
2. Find your planning batch (for example, **DEFAULT**)
3. Set the **Production Forecast** field to the forecast you want to plan against
4. Return to Planning Worksheet and click **Apply Forecast Filter** again

---

### Issue 2: "Lines I Expected to Stay Were Removed"

**Cause:** The item's forecast entries may all have quantity zero, or entries exist under a different forecast name.

**Solution:**
1. Open **Demand Forecasts** and search for the forecast configured on your batch
2. Filter forecast entries by the item number in question
3. Confirm that at least one entry has **Forecast Quantity** > 0
4. If all entries are zero, update the forecast with correct quantities and re-run Calculate Plan, then click Apply Forecast Filter again
5. If entries are under a different forecast name, update the **Production Forecast** field in Req. Wksh. Names to match

---

### Issue 3: "Lines I Expected to Be Removed Are Still on the Worksheet"

**Cause:** The item has at least one forecast entry with quantity > 0 in the configured forecast (even if small or far future), or the line is an MRP line (which is never removed).

**Solution:**
1. Open the forecast and search for the item
2. If the entry should not be there, remove it or set quantity to zero, then re-run Calculate Plan and click Apply Forecast Filter again
3. If the entry is intentional, the line is correct to remain—review and delete manually from worksheet if it should not create an order
4. If the line is an MRP line (component/purchase suggestion), it is intentionally kept—the filter only removes MPS lines

## FAQ

**Q: Does clicking Yes permanently delete anything from my forecast?**

A: No. The filter only removes planning lines from the Planning Worksheet. Forecast entries, items, and all other data remain untouched. Planning lines are temporary suggestions and can be regenerated at any time by running Calculate Regenerative Plan again.

---

**Q: Can I undo the removal after clicking Yes?**

A: No. Once lines are removed, they are gone from the current worksheet. To get them back, run Calculate Regenerative Plan again. If unsure, click **No** and manually delete the lines you do not need.

---

**Q: What forecast does the filter use?**

A: The filter uses the forecast set in the **Production Forecast** field on your planning batch in **Req. Wksh. Names**. Only that single forecast is checked. Items are kept only if they have a positive quantity in that forecast.

---

**Q: I ran Calculate Plan and my worksheet has too many lines. Can I still use this filter?**

A: Yes—as long as the **Production Forecast** field is set on your batch in Req. Wksh. Names, you can click **Apply Forecast Filter** at any time after running Calculate Plan.

---

**Q: Does the filter consider forecast quantities by date or location?**

A: No. The filter checks whether the item has any forecast entry with positive quantity in the selected forecast, regardless of date or location. If an item has even one qualifying entry, all of its planning lines are kept.

---

**Q: What does "zero-qty entries don't count" mean?**

A: Some forecasts include placeholder rows with quantity zero (items being phased out or not yet confirmed). The Forecast Filter treats these the same as having no forecast entry—if every entry for an item is zero, that item's planning lines will be removed.

---

**Q: Will this affect MRP lines (component purchase suggestions)?**

A: No. The Forecast Filter only evaluates MPS lines—production order suggestions for finished goods. MRP lines (component demand, purchase order suggestions for pots, cuttings, soil, etc.) are never removed by this filter.

---

## Best Practices

✅ **DO:**
- Set the Production Forecast field before starting planning sessions
- Run Apply Forecast Filter immediately after Calculate Regenerative Plan
- Review the removal summary to confirm expected results
- Keep forecast entries updated with realistic quantities
- Use filter when switching between seasonal forecasts

❌ **DON'T:**
- Forget to set the Production Forecast field (filter will error)
- Click Yes without reviewing which items will be removed
- Keep zero-quantity placeholder entries if they should be removed
- Assume the filter removes MRP lines (it only removes MPS lines)
- Manually re-add lines removed by filter without updating forecast first
