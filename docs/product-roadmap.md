## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the site broken; each phase ends with a demoable wiki state.
2. **Infrastructure before features.** The first phase sets up Quartz, build scripts, and deployment assumptions so later work is clean.
3. **Magic moment first.** The core reader path is SSO access, search, and a useful linked note.
4. **Test as you go.** Every phase includes local build verification and access or content checks.
5. **Progressive enhancement.** Start with a small pilot corpus, then add stricter audits and polish.
6. **Review before proceeding.** Push each completed phase as a PR and review before building on it.
7. **Protect the source boundary.** Readers get the website, not the private GitHub repo.
